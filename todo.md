## Критичные замечания / Critical Issues

### Безопасность
- [ ] **Удалить `.env` из репозитория** — файл содержит `DATABASE_URL` с реальным путём, добавить в `.gitignore`
- [ ] **Убрать `ignoreBuildErrors: true`** в `next.config.ts:7` — скрывает ошибки типизации при сборке
- [ ] **Включить `reactStrictMode: true`** в `next.config.ts:9` — сейчас отключён, что маскирует потенциальные баги
- [ ] **Включить `noImplicitAny: true`** в `tsconfig.json:14` — сейчас `false`, что снижает качество типизации
- [ ] **Валидация на серверном прокси** — `src/app/api/grafinya/proxy/route.ts:12` принимает произвольный `baseUrl` без валидации (SSRF-вектор)
- [ ] **Прокси не проверяет Content-Type ответа** — может упасть на не-JSON ответах (route.ts:42)
- [ ] **Access token передаётся в теле POST-запроса** через прокси — лучше через заголовки Cookie/HttpOnly

### База данных
- [ ] **Prisma schema не используется** — `schema.prisma` содержит шаблон `User`/`Post`, но проект работает через REST API к Графине. Либо интегрировать БД, либо удалить Prisma
- [ ] **`db/custom.db` закоммичен** в репозиторий — это файл БД, он не должен быть в VCS

## Качество кода / Code Quality

### Архитектура
- [ ] **page.tsx монолит** — `src/app/page.tsx` (541 строка) содержит header, sidebar, mobile menu, keyboard shortcuts. Разделить на подкомпоненты
- [ ] **Отсутствие серверных компонентов** — вся страница помечена `"use client"`. Next.js 16 поддерживает Server Components, стоит использовать
- [ ] **Нет разделения на feature-модули** — все компоненты лежат в одном `src/components/`. Предложить структуру:
  ```
  src/features/
    dashboards/
    explorer/
    datasources/
    settings/
    ...
  ```
- [ ] **`GraphinyaClient` синглтон** — `src/lib/grafinya-api.ts:467-478` использует глобальную переменную, что проблематично в SSR
- [ ] **Нет Zod-валидации** — `zod` установлен, но нигде не используется для валидации форм и API-ответов
- [ ] **Нет React Query / SWR** — ручное управление `isLoading` + `useState` вместо кэширования и рефетчинга через `@tanstack/react-query`

### Типизация
- [ ] **`noImplicitAny: false`** в tsconfig — много `any` в коде проходит без ошибок
- [ ] **Отсутствуют типы для API-ответов** —许多 интерфейсы определены, но API-маршруты не типизированы строго
- [ ] **`moduleData?: Record<string, unknown>`** в `Widget` типе — слишком широкий тип, стоит уточнить
